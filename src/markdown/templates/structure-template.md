## 代码组织原则

1. **单一职责**：每个文件应有明确且唯一的职责
2. **模块化**：代码应组织为可复用的模块
3. **可测试性**：结构设计应便于测试
4. **一致性**：遵循代码库中已建立的模式

### 示例结构：
```

└── dashboard/          # 自包含仪表盘子系统

    ├── shared/        # 共享类型/工具
```

### 关注点分离
- 仪表盘与核心业务逻辑隔离
- 拥有独立的 CLI 入口点以便独立运行
- 对主应用的依赖最小化
- 可在不影响核心功能的前提下禁用
